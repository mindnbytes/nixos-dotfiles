# Beszel Setup Notes

This document records the imperative setup steps required around the declarative NixOS configuration in `beszel.nix`.

Most of Beszel is configured declaratively, but the initial hub account, agent authorization key, and agent registration are created through the web interface and local files.

## Initial setup

1. Enable only the Beszel hub and its Caddy virtual host first.

2. Make the local hostname resolve from the client machine.

   Until proper local DNS is available, add this entry to `/etc/hosts`:

   ```text
   <server-lan-ip> monitor.home.arpa
   ```

3. Rebuild the server:

   ```fish
   sudo nixos-rebuild switch --flake .#nixos-btw
   ```

4. Check that the hub and Caddy are running:

   ```fish
   systemctl status beszel-hub caddy
   journalctl -u beszel-hub -u caddy -b
   ```

5. Open the Beszel hub:

   ```text
   http://monitor.home.arpa
   ```

   Create the first Beszel administrator account.

   The hub account, configuration, alerts, registered systems, and historical metrics are stored under:

   ```text
   /var/lib/beszel-hub
   ```

6. In the Beszel web interface, select **Add System** and configure:

   ```text
   Name: nixos-btw
   Host: 127.0.0.1
   Port: 45876
   ```

   Beszel will generate an SSH public key for the agent.

7. Create the agent environment file on the server:

   ```fish
   sudo install -d -m 700 /var/lib/secrets
   sudoedit /var/lib/secrets/beszel-agent.env
   ```

   Add the complete key generated by the hub:

   ```text
   KEY="ssh-ed25519 AAAA...complete-generated-key..."
   ```

   Keep the quotation marks because the SSH public key contains spaces.

   Protect the file:

   ```fish
   sudo chown root:root /var/lib/secrets/beszel-agent.env
   sudo chmod 600 /var/lib/secrets/beszel-agent.env
   ```

   Do not place the key directly in the Nix configuration. Ordinary Nix values are copied into the world-readable `/nix/store`.

8. Enable the Beszel agent declaration and rebuild:

   ```fish
   sudo nixos-rebuild switch --flake .#nixos-btw
   ```

9. Verify the agent:

   ```fish
   systemctl status beszel-agent
   journalctl -u beszel-agent -b
   ss -ltnp | grep 45876
   ```

   Expected listener:

   ```text
   127.0.0.1:45876
   ```

   The agent port should not be exposed through the firewall because the hub and agent run on the same server and communicate through localhost.

10. Return to the Beszel web interface and finish adding the system.

    The system should become green and begin displaying metrics.

## Backup and recovery

Include the hub state in the Borg backup:

```text
/var/lib/beszel-hub
```

Also back up the agent environment file through an appropriate secrets-backup process:

```text
/var/lib/secrets/beszel-agent.env
```

Do not commit the environment file or its key to the dotfiles repository.

If the agent environment file is lost, generate or retrieve a new agent key from the Beszel hub and recreate:

```text
/var/lib/secrets/beszel-agent.env
```

If the hub state is lost, the administrator account, systems, alerts, and historical metrics must either be restored from backup or configured again.

## Current intentional limitations

Do not open ports `8090` or `45876` in the firewall:

- `8090`: Beszel hub, accessed only through Caddy.
- `45876`: Beszel agent, accessed only through localhost.

Do not add `/mnt/backup` as an extra filesystem until it has been verified that Beszel polling does not trigger the systemd automount or wake the external HDD.

Do not enable SMART monitoring for the external backup disk until its effect on disk sleep and wake-up behaviour has been tested.

## Useful checks

Check all related services:

```fish
systemctl status beszel-hub beszel-agent caddy
```

Inspect logs from the current boot:

```fish
journalctl -u beszel-hub -u beszel-agent -u caddy -b
```

Test the Caddy virtual host locally:

```fish
curl -I -H 'Host: monitor.home.arpa' http://127.0.0.1
```

Check the Beszel listeners:

```fish
ss -ltnp | grep -E ':(8090|45876)\\b'
```

Expected network exposure:

```text
Caddy         LAN-facing on port 80
Beszel hub    localhost only on port 8090
Beszel agent  localhost only on port 45876
```
